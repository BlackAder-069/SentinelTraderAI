Data Ingestion is the foundation — you pull three streams simultaneously: OHLCV market data (Alpaca, Polygon.io, or Yahoo Finance for free-tier), news & SEC filings, and social sentiment. These need to be normalized into a common timestamped format.
Signal Processing is where raw data becomes actionable signals. Technical indicators (RSI, MACD, EMA) run on the price feed. An NLP model (FinBERT is purpose-built for this) scores news sentiment per ticker. Momentum scoring aggregates price velocity and volume surges into a single signal strength score.
LLM Agent Brain is the "agentic" part — this is where LangGraph or CrewAI orchestrates a ReAct loop. The agent has tools: it can look up signals, query a RAG store of historical earnings/macro context, and maintain memory across a trading session. It reasons over all inputs and decides: buy / sell / hold / size.
Risk Management acts as a hard gate before any order leaves the system. It enforces position sizing rules (Kelly criterion or fixed % of capital), stop-loss levels (ATR-based trailing stops are robust), and circuit breakers that halt the bot entirely if daily drawdown exceeds a threshold. This layer should be deterministic — no LLM involved.
Execution talks to the broker API. Alpaca is the easiest starting point (free paper trading, REST + WebSocket, Python SDK). Always build and test in paper mode first.
